## ❌ Gaps

### 1. Generator Uses Single Schema for All Types

**Location:** `lib/ai-generator.ts` → `generateCoreData()`

- Uses `PASS1_PROMPT` + `PASS1_SCHEMA` (symptom-focused: causes, repairs, diagnostic_steps) for **every** page type
- Does **not** call `getPromptByPageType(pageType)` or `composePromptForPageType(pageType)`
- Cause, repair, component pages need different output shapes (e.g. cause: explanation, affected_symptoms; repair: steps, tools)

**Fix:** Wire `composePromptForPageType(pageType)` and page-type-specific schemas into `generateCoreData`.

---

### 2. Thin Content Guards — Symptom Only

**Location:** `lib/diagnostic-engine.ts`

- `shouldGeneratePage(symptomId)` — requires `symptom.causes.length >= 2`
- `canBuildFromGraph(symptom)` — requires min 2 causes + 2 repairs
- **No guards** for: cause, repair, condition, component, system, diagnostic

**Risk:** Cause/repair pages could be generated with minimal or no linked data.

**Fix:** Add guards per type, e.g.:
- Cause: require linked symptom + ≥1 repair
- Repair: require linked cause or component
- Component: require ≥1 symptom or repair

---

### 3. Worker Queue Heavily Restricted

**Location:** `scripts/generation-worker.ts`

```sql
AND proposed_slug IN ('diagnose-ac-blowing-warm-air', 'why-does-capacitor-fail')
```

- Only 2 slugs are processed
- Cause, repair, condition, etc. are never generated by the worker in practice

---

### 4. Canary/Core Path Only for Symptom/Diagnose

- When `USE_CANARY=false` or no graph data: falls through to `generatePageContent`
- `generatePageContent` uses the same symptom-focused schema for all types
- Two-stage generator (`generateCoreOnlyPage`, `generateTwoStagePage`) is **symptom-only** — no cause/repair variants

## Recommended Fixes

1. **Wire page-type prompts:** Use `composePromptForPageType(pageType)` in `generateCoreData` instead of `PASS1_PROMPT`.
2. **Add page-type schemas:** Define PASS1-style schemas per type (cause, repair, component, etc.) and select by `pageType`.
3. **Add thin-content guards:** Implement `shouldGenerateCause`, `shouldGenerateRepair`, etc., and call before generation.
4. **Expand worker queue:** Remove or parameterize the slug filter so cause/repair/condition pages can be queued and generated.
